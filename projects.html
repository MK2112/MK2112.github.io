Conflicting Few-Shots

January 2026 • NLP

We examine how LLMs handle prompts with semantically conflicting examples, finding that recent models respond in very different ways. Some ignore the conflicts, some rely on positional biases, and others dynamically re-evaluate information based on where and how often conflicts appear. Model size and baseline accuracy do not reliably predict these behaviors. In some cases, well-placed inconsistencies can even improve performance, highlighting the importance of careful, model-specific prompt engineering.

DreamStride

May 2025 • Robotics

A modular implementation of the DayDreamer reinforcement learning algorithm for training a simulated Spot robot in WeBots. It recreates the walking dog experiment from DayDreamer, using socket-based communication to bridge perception, action, and learning. This makes Dreamer’s real-world efficiency accessible in a controlled, reproducible simulation environment that can be scaled.

MiniCorpus

February 2025 • NLP

Investigating, reproducing and improving MiniPile, a distilled subset of The Deduplicated Pile. Introducing several sampling techniques, like size-density-proportionate sampling, MiniCorpus creates training datasets that are smaller than MiniPile while maintaining competitive performance. The project provides a complete pipeline for dataset distillation, from embedding generation to cluster-based sampling, with comprehensive benchmarking across multiple sampling strategies and sketches theoretical ideas too.

MobileYOLOv3

October 2024 • Vision

A tiny, back-to-back implementation of the YOLOv3 object detection algorithm in PyTorch. Its aim is to showcase an entire training pipeline, from data preparation to model inference, in a single Jupyter Notebook. The model is trained on the ICDAR2015 dataset and gets one-shot pruned and quantized, making it suitable for use cases like real-time text detection on resource-constrained devices.

GPTemplate

January 2024 • NLP

A fully trainable and usable Generative Pre-Trained Transformer (GPT) within a Jupyter Notebook. I built this as a progression beyond Andrej Karpathy's Neural Networks Series to showcase GPT's implementation efficiency and basic inner workings. Users only need to provide a text file. GPT internalizes the textual patterns therein and generates contextually aligned text, given a prompt. The hugely popular GPT approach to NLP is thus made a bit more accessible.

Any_To_Any.py

November 2023 • Preprocessing

Media file conversion is a key requirement for data processing at scale. To address this, I built a lightweight Python interface for FFMPEG, simplifying conversion across a wide range of formats, from audio (MP3, FLAC,...) and images (PNG, GIF, BMP,...) to video (MP4, WebM, AVI,...), all through CLI or a Web Interface.
